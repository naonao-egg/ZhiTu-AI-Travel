## 一、 项目简介

### 1.1 什么是智途 (ZhiTu)
“智途”是一款专门针对现代旅行者“攻略制作繁琐”、“旅行信息碎片化”、“需求难以个性化”等痛点设计的智能化旅行管家与一站式路线生成系统。

在传统的旅行规划流程中，用户面临着“多平台查攻略耗时”、“景点距离难以把控”、“当地气象变化莫测”等核心痛点。本系统通过创新的**端到端 AI 驱动架构**，将大语言模型推理引擎、多模态实景图库检索、全球气象局真实数据源以及基于 Spring AI 的函数调用（Function Calling）完美融合。

用户只需通过极其自然的流式对话，系统便可自动感知用户情绪，分析其出游意图，并瞬间生成结构化的高保真行程卡片；同时，内置的**长期偏好记忆大脑**能够真正做到“千人千面”，让每一次的旅行推荐都极度契合用户的私域习惯与审美。

## 二、 系统架构与流向底座

### 2.1 拓扑双端解耦架构
本系统彻底摒弃了传统的单体结构设计，将核心业务逻辑、数据库持久化与长文本 AI 推理计算进行严密的物理分流：

1. **前端应用 (SPA)**：采用 React 18 + Vite 驱动，引入 Tailwind CSS 构建极具未来感的 UI 界面。专门负责轻量化组件渲染、流式消息装配与高颜值卡片排版。
2. **Java 业务与中转后端**：基于 Spring Boot 3.5.14，承担高频的数据中转、API 聚合以及 AI 调度。作为“微网关”，将前端的请求安全地转发至云端大模型底座。
3. **记忆持久化层**：采用 H2 嵌入式关系型文档数据库，无缝存储复杂的用户行程对象快照及长期记忆切片。
4. **AI 认知底座**：接入智谱 AI (ZhipuAI / GLM-4) 开放平台大模型，配合强大的系统提示词与 Function Calling 函数注册矩阵，充当整个项目的智慧大脑。

## 三、 环境依赖与安装部署

### 3.1 基础依赖环境要求
在准备进行本地编译和运行前，请务必保证您的宿主机已经正确安装并配置好了以下软件底座：

| 环境组件名称 | 推荐标准版本 | 环境变量与校验命令 |
| --- | --- | --- |
| **Java (JDK)** | OpenJDK 25 | `java -version` (确保运行环境无误) |
| **Node.js 运行时** | 20.15.0 (LTS) | `node -v` (用于前端轻量级构建与运行) |
| **项目构建工具** | Maven 3.9+ | 可直接使用项目内置的 `mvnw` 脚本 |

### 3.2 源码提取与启动流

**Step 1: 拉取项目**
请使用标准的 Git 终端拉取项目至本地：
```bash
git clone https://github.com/您的用户名/ZhiTu-AI-Travel.git
cd ZhiTu-AI-Travel
```

**Step 2: 启动底层数据中心与 Java 后端**
打开第一个终端窗口，拉起 Spring Boot 核心服务（内置 H2 将自动启动建表）：
```bash
cd 智途/源码/后端
# Windows 环境
.\mvnw.cmd spring-boot:run
# macOS/Linux 环境
./mvnw spring-boot:run
```
*预期日志：`Tomcat started on port 8080 (http)`*

**Step 3: 拉起前端轻量级热重载客户端**
打开第二个独立的终端窗口：
```bash
cd 智途/源码/前端
npm install
npm run dev
```
*预期日志：`VITE ready in X ms -> Local: http://localhost:3000/`*

## 四、 配置说明详解

### 4.1 Spring Boot 服务端高敏配置 (application.properties)
在 `后端/src/main/resources/` 目录下，找到并对 `application.properties` 进行精准配置。这里管理着接入大模型和数据库的核心路由：

```properties
# Web 服务占用端口，前端连接需直接向此端口发起建立
server.port=8080

# 智谱 AI 性能大模型底层云对齐参数
# 您的开发者账户密钥令牌，确保 sk- 开头
spring.ai.zhipuai.api-key=替换为您申请的ZhipuAI_API_KEY
# 指定调用的基础推理大模型。系统采用指令精调版，平衡了控糖医疗语义及响应时间
spring.ai.zhipuai.chat.options.model=glm-4

# H2 内存级关系型数据库连接字符串
spring.datasource.url=jdbc:h2:file:./data/smarttrek_db
spring.datasource.driverClassName=org.h2.Driver
```

### 4.2 外部微服务 API 密匙管理 (AiTools.java)
和风天气与无版权实景图库的 API 凭据，已在 Spring 容器初始化时由 `AiTools.java` 安全加载。如有更换需求，可在此类中直接替换常量：

### 5.1 全局旅行上下文交互 (ExplorePage)
打开浏览器访问 `http://localhost:3000`，系统将呈现赛博极简风格的智途对讲机视窗。
- **用户行为**：点击底部的输入框，键入“我想去西安玩”。此时，AI 后端将触发天气探测（Weather Tool），数秒内除了呈现详尽的景点卡片，页面还会优雅地弹出一个蓝色的**西安专属天气组件**，精确呈现当前温湿度及穿衣防晒建议。

为了给用户带来极致的定制化体验，系统引入了不可思议的零门槛“长期记忆”系统。

- **主动管理**：点击主界面顶部的紫色“**记忆碎片**”按钮，用户可以直接查看到智途为自己记录的所有画像特征。点击“✖️”即可让 AI 彻底遗忘某条记录。跨越新会话后，系统将自动读取这些记忆，并在规划时严格执行。

## 六、 安全屏障与系统防护机制

### 6.1 CORS 跨域精密控制
由于系统采用多端口解耦微服务设计，React 前端 (3000) 在向 Java 后端 (8080) 发起请求时，必然会触发浏览器的同源策略限制。系统在 Spring 层面配置了精细化的全局跨域过滤器，并严格限制了 Headers 的吞吐范围，筑牢了系统的网络防御防线。

### 6.2 敏感大模型 API 密钥内网封装隔离
市面上许多轻量级前端应用为了图方便，直接在前端 Axios 请求中携带 `sk-xxx` 密钥调用大模型接口。这种做法极其危险。
**智途系统安全隔离设计**：将核心的 api-key 和底层路由完全固化在 Spring Boot 的内网物理服务端。前端 React 只需要与本地的 8080 端口建立普通授信连接，完全接触不到真实的第三方云端授权密钥。这种彻底的业务解耦完美隔绝了安全隐患。

### 6.3 极端数据与 GZIP 网络报文熔断
由于外部天气 API 返回的往往是经过极高压缩比的二进制 GZIP 数据。如果系统直接使用原生解析器进行解析，将会直接导致进程 OOM 甚至系统内核崩溃。
系统底层在 `fetchAndDecompress` 中引入了基于 Magic Number `0x1F8B` 的前置二进制嗅探，成功分离 GZIP 流，并在后端优雅解压为明文 JSON。

## 八、 常见问题与排产故障排除 (FAQ)

**Q1：网页加载出来了，但是发消息后一直提示“系统开小差了”？**
- 根本原因分析：大模型 API 密钥欠费、被封禁，或者后端 Java 服务未正常启动。
- 解决方案：请在终端检查 `mvnw spring-boot:run` 进程是否正在运行；检查 `application.properties` 中的 API KEY 是否为合法有效值。

**Q2：运行 Java 时提示 `java.lang.UnsupportedClassVersionError`？**
- 根本原因分析：JDK 版本不匹配。本项目编译采用了 JDK 25 的最新语言特性。
- 解决方案：请在终端输入 `java -version`。如果不等于 25 或以上，请前往 Oracle 或 Adoptium 官网下载最新版 JDK 并配置环境变量。
